# 接口（Interface）深入

## 1. 接口 vs 类型别名

| 特性 | `interface` | `type` |
|------|-------------|--------|
| 声明合并 | ✅ 同名接口自动合并 | ❌ 同名 type 会报错 |
| 扩展方式 | `extends` | `&`（交叉类型） |
| 可描述的类型 | 仅对象结构 | 对象、联合、元组、字面量等 |
| 性能 | 更快（被缓存） | 略慢（每次计算交叉） |
| 推荐场景 | 对象类型、公开 API | 联合类型、元组、映射类型 |

```typescript
// interface —— 只能描述对象结构
interface User {
  name: string
  age: number
}

// type —— 可以描述任意类型
type ID = string | number
type Point = [number, number]
type Callback = (data: string) => void
```

## 2. 接口声明合并

同名接口会自动合并，这是 `type` 做不到的：

```typescript
interface Animal {
  name: string
}

interface Animal {
  age: number
}

// 最终 Animal 同时拥有 name 和 age
const cat: Animal = {
  name: 'Tom',
  age: 2
}
```

这在「扩展第三方类型」时特别有用，比如给 `Window` 添加自定义属性：

```typescript
interface Window {
  myCustomProp: string
}

// 现在 window.myCustomProp 有类型了
console.log(window.myCustomProp)
```

## 3. 接口继承（extends）

### 单继承

```typescript
interface Animal {
  name: string
}

interface Dog extends Animal {
  breed: string
}

const dog: Dog = {
  name: 'Buddy',
  breed: 'Golden Retriever'
}
```

### 多继承

```typescript
interface Named {
  name: string
}

interface Aged {
  age: number
}

interface Person extends Named, Aged {
  email: string
}

const p: Person = {
  name: 'yzb',
  age: 18,
  email: 'yzb@example.com'
}
```

### 继承类

接口可以继承类的类型（不包括实现）：

```typescript
class Point {
  x: number = 0
  y: number = 0
}

interface Point3D extends Point {
  z: number
}

const p3: Point3D = { x: 1, y: 2, z: 3 }
```

## 4. 函数接口

```typescript
// 定义函数类型
interface GreetFn {
  (name: string): string
}

const greet: GreetFn = (name) => `Hello, ${name}`

// 带属性的函数（如 jQuery 的 $）
interface JQueryFn {
  (selector: string): HTMLElement
  version: string
}

// 实现时可以用类型断言来满足接口
const $ = ((selector: string) => {
  return document.querySelector(selector)!
}) as JQueryFn
$.version = '1.0.0'
```

## 5. 索引签名

### 字符串索引

```typescript
interface StringMap {
  [key: string]: string
}

const colors: StringMap = {
  red: '#FF0000',
  green: '#00FF00',
  blue: '#0000FF'
}
```

### 数字索引

```typescript
interface NumberArray {
  [index: number]: string
}

const arr: NumberArray = ['a', 'b', 'c']
console.log(arr[0]) // 'a'
```

### 混合索引

```typescript
interface Mixed {
  [key: string]: number    // 字符串索引返回值必须兼容数字索引的返回值
  [index: number]: string  // ❌ 错误！string 不兼容 number
}

// 正确的做法：数字索引返回值必须是字符串索引返回值的子类型
interface MixedCorrect {
  [key: string]: number | string
  [index: number]: number  // number 是 number|string 的子类型 ✅
}
```

## 6. 接口中的只读与可选

```typescript
interface Config {
  readonly host: string       // 只读，赋值后不可修改
  readonly port: number
  timeout?: number            // 可选
  retries?: number
}

const config: Config = {
  host: 'localhost',
  port: 3000
}

// config.host = 'changed' // ❌ 报错，只读属性不能修改

// 利用 Readonly 工具类型
type ReadonlyConfig = Readonly<Config>
```

## 7. 接口描述类结构

```typescript
interface ClockInterface {
  currentTime: Date
  setTime(d: Date): void
}

class Clock implements ClockInterface {
  currentTime: Date = new Date()

  setTime(d: Date) {
    this.currentTime = d
  }

  constructor(h: number, m: number) {}
}
```

### 类静态部分与实例部分

`implements` 只检查**实例部分**，不检查静态部分：

```typescript
// 构造签名 —— 描述构造函数
interface ClockConstructor {
  new (hour: number, minute: number): ClockInterface
}

// 实例接口
interface ClockInterface {
  tick(): void
}

function createClock(
  ctor: ClockConstructor,
  hour: number,
  minute: number
): ClockInterface {
  return new ctor(hour, minute)
}

class DigitalClock implements ClockInterface {
  constructor(h: number, m: number) {}
  tick() {
    console.log('beep beep')
  }
}

class AnalogClock implements ClockInterface {
  constructor(h: number, m: number) {}
  tick() {
    console.log('tick tock')
  }
}

// 使用
const digital = createClock(DigitalClock, 7, 32)
const analog = createClock(AnalogClock, 7, 32)
```

## 8. 混合类型

一个对象可以同时作为函数和对象使用：

```typescript
interface Counter {
  // 作为函数调用
  (start: number): string

  // 作为对象使用
  interval: number
  reset(): void
}

function createCounter(): Counter {
  const counter = ((start: number) => {
    console.log('counting from', start)
    return `started at ${start}`
  }) as Counter

  counter.interval = 1000
  counter.reset = () => {
    console.log('reset')
  }
  return counter
}

const c = createCounter()
c(10)            // 作为函数调用
c.reset()        // 调用方法
c.interval = 500 // 访问属性
```

## 9. 接口最佳实践

1. **优先使用 `interface` 声明对象类型**：可被 `extends`、可声明合并，适合公开 API。
2. **复杂类型用 `type`**：联合类型、元组、映射类型等。
3. **不要过度使用可选属性**：需要区分「属性不存在」与「属性值为 undefined」。
4. **用 `readonly` 保护不应被修改的数据**。
5. **利用声明合并在 `.d.ts` 中扩展第三方类型**。
