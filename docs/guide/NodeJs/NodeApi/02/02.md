# Koa 服务器

## 1. Koa 介绍

1. Koa 是 Express 原班人马打造的新一代 Node.js Web 框架，核心代码只有约 1600 行，更加轻量。
2. 相比于 Express，Koa 的核心设计理念是：

- **洋葱模型中间件**：通过 `async/await` 实现，不仅可以拦截请求，还可以拦截响应（Express 中间件只能单向流）。
- **不内置任何中间件**：请求体解析、路由、静态资源等都需要单独安装，项目体量完全由你控制。
- **更强的异步处理能力**：所有中间件天然支持 `async/await`，无需额外包来处理异步错误。

3. 安装 Koa：

```bash
npm init -y
npm i koa
```

## 2. Koa 基础使用

```js
const Koa = require('koa')
const app = new Koa()

// use 注册中间件
app.use(async (ctx, next) => {
  console.log('收到请求:', ctx.url)
  await next() // 让请求继续流向下一个中间件
  console.log('响应已发送') // next() 之后的代码在响应后执行（洋葱模型的回程）
})

// 处理具体路由
app.use(async (ctx) => {
  if (ctx.path === '/login' && ctx.method === 'GET') {
    ctx.body = 'Login Success'
  } else {
    ctx.body = 'Hello Koa'
  }
})

app.listen(8000, () => {
  console.log('Koa 服务已启动 → http://localhost:8000')
})
```

### ctx 上下文对象

1. Koa 将 `request` 和 `response` 合并到 `ctx`（上下文对象）中：

- `ctx.request` / `ctx.req`：Node 原生请求对象
- `ctx.response` / `ctx.res`：Node 原生响应对象
- `ctx.url` / `ctx.path`：请求路径（`ctx.path` 不含 query）
- `ctx.method`：请求方法
- `ctx.query`：查询参数对象
- `ctx.body`：设置响应体（自动根据类型设置 Content-Type）

2. `next` 本质上是一个 `dispatch` 函数，调用它会把请求传递给下一个中间件。如果不调用 `next()`，后续中间件不会执行。

### 洋葱模型

```js
const Koa = require('koa')
const app = new Koa()

app.use(async (ctx, next) => {
  console.log('1. 进入第 1 层')
  await next()
  console.log('5. 离开第 1 层')
})

app.use(async (ctx, next) => {
  console.log('2. 进入第 2 层')
  await next()
  console.log('4. 离开第 2 层')
})

app.use(async (ctx) => {
  console.log('3. 核心业务逻辑')
  ctx.body = 'Hello Onion Model'
})

// 输出顺序：
// 1. 进入第 1 层
// 2. 进入第 2 层
// 3. 核心业务逻辑
// 4. 离开第 2 层
// 5. 离开第 1 层

app.listen(8000)
```

洋葱模型的核心价值：第一个中间件可以**同时处理请求和响应**阶段，非常适合做统一的日志记录、请求计时、异常捕获等。

```js
app.use(async (ctx, next) => {
  const start = Date.now()
  try {
    await next()
  } catch (err) {
    console.error('捕获到错误:', err.message)
    ctx.status = 500
    ctx.body = '服务器内部错误'
  }
  const ms = Date.now() - start
  console.log(`${ctx.method} ${ctx.url} - ${ms}ms`)
})
```

## 3. Koa 路由

Koa 本身不内置路由，需要安装 `koa-router`：

```bash
npm i koa-router
```

**主文件 `index.js`**：

```js
const Koa = require('koa')
const userRouter = require('./router/user')

const app = new Koa()

app.use(userRouter.routes())
// allowedMethods：当路径匹配但 Method 不匹配时，返回 405 而不是 404
app.use(userRouter.allowedMethods())

app.listen(8000, () => {
  console.log('Koa 路由服务已启动')
})
```

**路由文件 `router/user.js`**：

```js
const Router = require('koa-router')

// prefix 会自动拼接到所有路径的前面
const router = new Router({ prefix: '/users' })

router.get('/', async (ctx) => {
  ctx.body = 'User List'
})

router.post('/', async (ctx) => {
  ctx.body = 'Create User'
})

router.put('/:id', async (ctx) => {
  // 获取路径参数
  console.log(ctx.params) // { id: '123' }
  ctx.body = `Update User ${ctx.params.id}`
})

module.exports = router
```

## 4. 请求参数解析

### URL 上的参数

```js
const Koa = require('koa')
const Router = require('koa-router')

const app = new Koa()
const userRouter = new Router({ prefix: '/users' })

// GET /users/123?name=yzb
userRouter.get('/:id', async (ctx) => {
  console.log('路径参数:', ctx.params) // { id: '123' }
  console.log('查询参数:', ctx.query)  // { name: 'yzb' }
  ctx.body = { params: ctx.params, query: ctx.query }
})

app.use(userRouter.routes())
app.listen(8000)
```

### 请求体参数解析

推荐使用 `koa-body`（它同时支持 JSON、urlencoded、multipart/form-data，包括文件上传）：

```bash
npm i koa-body
```

```js
const Koa = require('koa')
const { koaBody } = require('koa-body')

const app = new Koa()

// koaBody 统一处理各类请求体
app.use(koaBody({
  multipart: true, // 开启文件上传支持
  formidable: {
    maxFileSize: 10 * 1024 * 1024 // 限制文件最大 10MB
  }
}))

app.use(async (ctx) => {
  console.log('请求体:', ctx.request.body) // 普通字段
  console.log('上传文件:', ctx.request.files) // 文件

  ctx.body = {
    body: ctx.request.body,
    files: ctx.request.files
  }
})

app.listen(8000, () => {
  console.log('Koa 参数解析服务已启动')
})
```

> **说明**：旧版笔记中常用 `koa-bodyparser` + `koa-multer`，它们已分别被 `koa-body` 和 `@koa/multer` 取代。`koa-body` 可以一举处理 JSON、urlencoded、form-data 和文件上传，推荐统一使用。

### 各格式详细用法

**JSON 请求体**：

```js
const Koa = require('koa')
const { koaBody } = require('koa-body')
const Router = require('koa-router')

const app = new Koa()
app.use(koaBody())

const router = new Router({ prefix: '/api' })

router.post('/login', async (ctx) => {
  console.log(ctx.request.body) // { username: 'admin', password: '123456' }
  ctx.body = { code: 0, message: '登录成功' }
})

app.use(router.routes())
app.listen(8000)
```

**文件上传**：

```js
const Koa = require('koa')
const { koaBody } = require('koa-body')
const Router = require('koa-router')
const path = require('node:path')
const fs = require('node:fs')

const app = new Koa()
const uploadRouter = new Router({ prefix: '/upload' })

app.use(koaBody({
  multipart: true,
  formidable: {
    uploadDir: './uploads',              // 文件临时存放目录
    keepExtensions: true,                // 保留文件扩展名
    maxFileSize: 10 * 1024 * 1024        // 10MB 限制
  }
}))

uploadRouter.post('/avatar', async (ctx) => {
  const file = ctx.request.files.avatar // files 中的字段名与上传的 key 一致

  if (!file) {
    ctx.status = 400
    ctx.body = { error: '请选择文件' }
    return
  }

  // file 可能是数组（多文件）或单个对象（单文件）
  const f = Array.isArray(file) ? file[0] : file

  // 重命名文件（添加时间戳防止覆盖）
  const ext = path.extname(f.originalFilename)
  const newName = Date.now() + ext
  const newPath = path.join('./uploads', newName)
  fs.renameSync(f.filepath, newPath)

  console.log('文件已保存:', newPath)
  console.log('其他字段:', ctx.request.body)

  ctx.body = {
    code: 0,
    message: '上传成功',
    filename: newName,
    url: `/uploads/${newName}`
  }
})

app.use(uploadRouter.routes())
app.listen(8000, () => {
  console.log('Koa 文件上传服务已启动')
})
```

## 5. 响应内容

```js
const Koa = require('koa')
const app = new Koa()

app.use(async (ctx) => {
  // 设置状态码 —— 两种方式
  ctx.response.status = 404
  ctx.status = 200

  // 设置响应体 —— ctx.body 可以接受多种类型
  ctx.body = 'Hello Koa'                    // 字符串 → text/plain
  ctx.body = { name: 'yzb', age: 18 }       // 对象 → application/json
  ctx.body = ['a', 'b', 'c']                // 数组 → application/json
  ctx.body = Buffer.from('hello')           // Buffer → application/octet-stream

  // 设置响应头
  ctx.set('X-Custom-Header', 'value')
  ctx.type = 'application/json'
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

## 7. 错误处理

### 统一错误处理中间件

利用洋葱模型的特性，在最外层中间件中捕获所有错误：

```js
const Koa = require('koa')
const app = new Koa()

// 最外层 —— 统一错误捕获
app.use(async (ctx, next) => {
  try {
    await next()
  } catch (err) {
    console.error('服务器错误:', err.message)
    ctx.status = err.status || 500
    ctx.body = {
      error: err.message,
      code: ctx.status
    }
  }
})

app.use(async (ctx) => {
  // 模拟业务逻辑中的错误
  const isLogin = false
  if (!isLogin) {
    const err = new Error('您还没有登录')
    err.status = 401
    throw err
  }
  ctx.body = '欢迎回来'
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

### 通过 EventEmitter 处理错误

Koa 的 `app` 本身是一个 EventEmitter，可以监听 `error` 事件：

```js
const Koa = require('koa')
const app = new Koa()

app.use(async (ctx, next) => {
  const isLogin = false
  if (!isLogin) {
    // 抛出错误，同时发出 error 事件
    ctx.app.emit('error', new Error('未登录'), ctx)
    ctx.status = 401
    ctx.body = { error: '未登录' }
    return
  }
  await next()
})

// 集中监听 error 事件（日志、告警等）
app.on('error', (err, ctx) => {
  console.error('错误事件:', err.message, '—', ctx.url)
})

app.listen(8000)
```

## 8. Koa vs Express 对比

| 特性 | Express | Koa |
|------|---------|-----|
| 核心大小 | ~1.5 万行 | ~1600 行 |
| 中间件模型 | 线性（单向流） | 洋葱模型（双向，async/await） |
| 异步处理 | 需手动 try/catch | 天然支持 async/await |
| 内置功能 | 路由、静态资源等内置 | 零内置，全部按需安装 |
| 错误处理 | 4 参数中间件 `(err,req,res,next)` | `try/catch` + 洋葱模型外层捕获 |
| 路由 | 内置 | 需要 `koa-router` |
| 请求体解析 | `express.json()` 内置 | 需要 `koa-body` |
| 响应体 | `res.send()` / `res.json()` | `ctx.body = xxx` |
| 社区生态 | 最成熟，中间件最多 | 活跃，中间件较新 |
| 适用场景 | 任何规模项目 | 追求轻量、高性能的项目 |

## 9. Koa 常用中间件速查

| 中间件 | 功能 | 安装 |
|--------|------|------|
| `koa-router` | 路由管理 | `npm i koa-router` |
| `koa-body` | JSON/urlencoded/form-data/文件统一解析 | `npm i koa-body` |
| `koa-static` | 静态资源托管 | `npm i koa-static` |
| `koa-views` | 模板引擎渲染 | `npm i koa-views` |
| `koa-cors` | 跨域处理 | `npm i koa-cors` (或 `@koa/cors`) |
| `koa-compress` | Gzip 压缩响应 | `npm i koa-compress` |
| `koa-helmet` | 安全头 | `npm i koa-helmet` |
| `koa-morgan` | HTTP 请求日志 | `npm i koa-morgan` |
| `koa-session` | Session 管理 | `npm i koa-session` |
| `koa-jwt` | JWT 认证 | `npm i koa-jwt` |
| `@koa/multer` | 文件上传（multer 的 Koa 适配版） | `npm i @koa/multer multer` |

典型中间件组合：

```js
const Koa = require('koa')
const { koaBody } = require('koa-body')
const cors = require('@koa/cors')
const helmet = require('koa-helmet')
const compress = require('koa-compress')

const app = new Koa()

app.use(helmet())
app.use(cors())
app.use(compress())
app.use(koaBody({ multipart: true }))

// 业务路由 ...

app.listen(8000)
```

## 10. Koa 最佳实践

1. **充分利用洋葱模型**：在最外层中间件统一处理日志、计时、异常捕获，无需在每个路由中重复。
2. **所有中间件使用 `async/await`**：Koa 基于 async 构建，同步中间件可能导致潜在问题。
3. **用 `koa-body` 替代 `koa-bodyparser` + `koa-multer`**：一个包搞定所有请求体解析。
4. **始终在最外层做错误捕获**：利用洋葱模型，`try/catch` 包裹 `await next()` 即可。
5. **结构分层**：`router` 目录放路由，`controller` 目录放业务逻辑，`middleware` 目录放中间件。
6. **不要在中间件中忘记 `await next()`**：否则后续中间件不会执行，洋葱模型失效。
7. **生产环境用环境变量**：通过 `process.env` 管理敏感配置（数据库连接、密钥等）。
