# 创建 node 服务器

## 1. 使用 http 模块

1. Node.js 内置了 `http` 模块，可以直接用来创建一个 Web 服务器。
2. `http.createServer` 返回服务器实例，底层是通过 `new http.Server` 创建的。
3. 创建 `Server` 时需要传入一个回调函数（请求监听器），它有 2 个回调参数：

- `request`：请求对象，包含请求相关的信息（url、method、headers 等）。
- `response`：响应对象，包含我们要发送给客户端的信息。

```js
const http = require('node:http')
const HTTP_PORT = 8888

const server = http.createServer((request, response) => {
  response.end('Hello World')
})

server.listen(HTTP_PORT, () => {
  console.log('服务已启动 → http://localhost:' + HTTP_PORT)
})
```

`listen` 函数有三个参数：

- `port`：端口号，可以不传（系统自动分配），生产环境通常写入环境变量。
- `host`：主机地址，默认是 `0.0.0.0`。
  - `localhost`：本质上是一个域名，通常被解析成 `127.0.0.1`。
  - `127.0.0.1`：回环地址（Loopback），发出去的包在网络层直接被自己接收，不经过数据链路层和物理层。监听 `127.0.0.1` 时，同一网段下的其他主机无法通过 IP 访问。
  - `0.0.0.0`：监听 IPv4 上所有地址。监听 `0.0.0.0` 时，同一网段下的其他主机可以通过 IP 访问。
- 回调函数：服务器启动成功时的回调。

```js
server.listen(8888, '0.0.0.0', () => {
  console.log('服务已启动')
})
```

## 2. request 对象

在向服务器发送请求时，会携带很多信息例如 URL、请求方法、headers 等。这些信息被 Node 封装到 `request` 对象中。

```js
const http = require('node:http')

const server = http.createServer((req, res) => {
  console.log('请求地址:', req.url)
  console.log('请求方法:', req.method)
  console.log('请求头:', req.headers)
  res.end('Hello World')
})

server.listen(8888, () => {
  console.log('服务已启动')
})
```

## 3. URL 的处理

一般情况下服务器需要根据不同的请求地址，作出不同的响应。在 v24.x 中推荐使用内置的 `URL` 类来解析路径（旧版 `url.parse` 已不推荐使用）。

```js
const http = require('node:http')

const server = http.createServer((req, res) => {
  // 使用 URL 类解析
  const urlObj = new URL(req.url, `http://${req.headers.host}`)
  const pathname = urlObj.pathname // 路径部分，如 /login
  const searchParams = urlObj.searchParams // 查询参数

  if (pathname === '/login') {
    const username = searchParams.get('username')
    const password = searchParams.get('password')
    console.log('登录:', username, password)
    res.end('登录成功')
  } else if (pathname === '/users') {
    res.end('用户列表')
  } else {
    res.end('404 Not Found')
  }
})

server.listen(8888, () => {
  console.log('服务已启动')
})
```

> **注意**：旧版常用 `url.parse()` + `querystring.parse()`，现在推荐直接用 `URL` 类的 `searchParams` 来获取查询参数，更简洁直观。

## 4. Method 的处理

在 RESTful 规范中，数据的增删改查应通过不同的请求方式来区分：

- `GET`：查询数据
- `POST`：新建数据
- `PUT` / `PATCH`：更新数据
- `DELETE`：删除数据

```js
const http = require('node:http')

const server = http.createServer((req, res) => {
  const { pathname } = new URL(req.url, `http://${req.headers.host}`)

  if (pathname === '/users') {
    switch (req.method) {
      case 'GET':
        res.end('查询用户列表')
        break
      case 'POST':
        res.end('创建用户')
        break
      case 'DELETE':
        res.end('删除用户')
        break
      default:
        res.statusCode = 405
        res.end('Method Not Allowed')
    }
  } else {
    res.statusCode = 404
    res.end('Not Found')
  }
})

server.listen(8888, () => {
  console.log('服务已启动')
})
```

## 5. 获取请求体数据

当客户端通过 POST / PUT 发送 JSON 数据时，需要监听 `req` 的 `data` 和 `end` 事件来获取数据。

```js
const http = require('node:http')

const server = http.createServer((req, res) => {
  const { pathname } = new URL(req.url, `http://${req.headers.host}`)

  if (pathname === '/login' && req.method === 'POST') {
    // 设置字符编码（文本数据用 utf-8，二进制数据如图片则不需要设置）
    req.setEncoding('utf-8')

    // data 事件：每次收到一块数据时触发（分块传输时可能触发多次）
    let body = ''
    req.on('data', (chunk) => {
      body += chunk
    })

    // end 事件：数据接收完毕
    req.on('end', () => {
      try {
        const { username, password } = JSON.parse(body)
        console.log('收到登录请求:', username, password)
        res.setHeader('Content-Type', 'application/json')
        res.end(JSON.stringify({ code: 0, message: '登录成功' }))
      } catch {
        res.statusCode = 400
        res.end('请求数据格式错误')
      }
    })
  } else {
    res.statusCode = 404
    res.end('Not Found')
  }
})

server.listen(8888, () => {
  console.log('服务已启动')
})
```

### 文件上传（multipart/form-data）

文件上传的请求体格式为 `multipart/form-data`，需要用 `boundary` 分隔符来解析。下面是一个简化版的手动解析示例：

```js
const http = require('node:http')
const fs = require('node:fs')
const path = require('node:path')

const server = http.createServer((req, res) => {
  if (req.url === '/upload' && req.method === 'POST') {
    const contentType = req.headers['content-type']

    // 提取 boundary 分隔符
    const boundary = '--' + contentType.split('boundary=')[1]

    const chunks = []
    req.on('data', (chunk) => chunks.push(chunk))
    req.on('end', () => {
      const body = Buffer.concat(chunks)

      // 按 boundary 分割各部分
      const parts = body.toString('binary').split(boundary)

      for (const part of parts) {
        if (part.includes('filename=')) {
          // 这是文件部分 —— 分离头部和文件内容
          const [headerStr, fileContent] = part.split('\r\n\r\n')

          // 从头部提取文件名
          const filenameMatch = headerStr.match(/filename="(.+?)"/)
          if (filenameMatch) {
            const filename = filenameMatch[1]
            // 去掉末尾的 \r\n
            const cleanContent = fileContent.substring(0, fileContent.lastIndexOf('\r\n'))
            fs.writeFileSync(
              path.join(__dirname, 'uploads', filename),
              cleanContent,
              'binary'
            )
            console.log('文件已保存:', filename)
          }
        }
      }

      res.end('文件上传成功')
    })
  } else {
    res.statusCode = 404
    res.end('Not Found')
  }
})

server.listen(8888, () => {
  console.log('文件上传服务已启动')
})
```

> **提示**：实际项目中的文件上传通常不手写解析逻辑，而是使用 `multer`（Express）或 `@koa/multer`（Koa）等成熟中间件。

## 6. headers 常用属性

请求头包含大量上下文信息，以下是常见字段：

| 字段 | 含义 |
|------|------|
| `content-type` | 请求体格式：`application/json`、`text/plain`、`multipart/form-data` 等 |
| `content-length` | 请求体的大小（字节） |
| `accept` | 客户端可接受的响应格式 |
| `accept-encoding` | 客户端支持的压缩格式，如 `gzip`、`br` |
| `user-agent` | 客户端信息（浏览器、操作系统等） |
| `connection` | `keep-alive` 表示复用 TCP 连接 |

### keep-alive 机制

- HTTP/1.0：默认每次请求结束后断开连接。如需保持，浏览器需加 `Connection: keep-alive`。
- HTTP/1.1：默认启用 `Connection: keep-alive`，Node 中默认保持 5 秒。
- 在连接保持期间，后续请求可复用同一个 TCP 连接，减少握手开销。

### accept-encoding

告知服务器客户端支持的文件压缩格式。例如 Webpack 可配置 gzip 压缩 JS/CSS 文件，服务器返回 `.gz` 文件以减小传输体积。

## 7. response 处理

### 返回数据

- `response.write(data)`：写入数据，不关闭流，可多次调用。
- `response.end([data])`：写入最后数据并关闭流。**必须调用 `end`**，否则客户端会一直等待直到超时。

```js
const http = require('node:http')

const server = http.createServer((req, res) => {
  res.write('Hello ')
  res.write('World')
  res.end('!')
  // 客户端收到: Hello World!
})

server.listen(8888)
```

### 设置状态码

```js
// 方式一：直接赋值
res.statusCode = 404

// 方式二：writeHead 同时设置状态码和响应头
res.writeHead(200, {
  'Content-Type': 'application/json;charset=utf-8'
})
res.end(JSON.stringify({ code: 0 }))
```

### 设置响应头

```js
// 方式一：逐个设置
res.setHeader('Content-Type', 'application/json;charset=utf-8')
res.setHeader('X-Powered-By', 'Node.js')
res.statusCode = 200

// 方式二：writeHead 同时设置状态码和响应头
res.writeHead(200, {
  'Content-Type': 'application/json;charset=utf-8',
  'X-Powered-By': 'Node.js'
})

res.end(JSON.stringify({ code: 0 }))
```

## 8. 使用 http 模块发送请求

Node.js 也可以作为客户端向其他服务器发送请求。

### 发送 GET 请求

```js
const http = require('node:http')

http.get('http://localhost:8888/api/users', (res) => {
  let data = ''
  res.on('data', (chunk) => { data += chunk })
  res.on('end', () => {
    console.log('响应数据:', data)
  })
}).on('error', (err) => {
  console.error('请求失败:', err.message)
})
```

### 发送 POST 请求

```js
const http = require('node:http')

const postData = JSON.stringify({ username: 'admin', password: '123456' })

const options = {
  hostname: 'localhost',
  port: 8888,
  path: '/api/login',
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Content-Length': Buffer.byteLength(postData)
  }
}

const req = http.request(options, (res) => {
  let data = ''
  res.on('data', (chunk) => { data += chunk })
  res.on('end', () => {
    console.log('响应数据:', data)
  })
})

req.on('error', (err) => {
  console.error('请求失败:', err.message)
})

// 写入请求体并结束
req.write(postData)
req.end()
```

### 发送 HTTPS 请求

```js
const https = require('node:https')

https.get('https://api.example.com/data', (res) => {
  let data = ''
  res.on('data', (chunk) => { data += chunk })
  res.on('end', () => {
    console.log('响应数据:', JSON.parse(data))
  })
}).on('error', (err) => {
  console.error('请求失败:', err.message)
})
```

## 9. 综合实战案例

### JSONP 跨域

```js
const http = require('node:http')

const server = http.createServer((req, res) => {
  const urlObj = new URL(req.url, `http://${req.headers.host}`)
  const callbackName = urlObj.searchParams.get('callback')

  if (urlObj.pathname === '/api/users' && callbackName) {
    const data = [
      { name: '张三', age: 18 },
      { name: '李四', age: 20 }
    ]
    // JSONP 原理：返回一个函数调用，参数是 JSON 数据
    res.setHeader('Content-Type', 'application/javascript')
    res.end(`${callbackName}(${JSON.stringify(data)})`)
  } else {
    res.statusCode = 404
    res.end('Not Found')
  }
})

server.listen(3000, () => {
  console.log('JSONP 服务已启动')
})
```

### CORS 跨域

```js
const http = require('node:http')

const server = http.createServer((req, res) => {
  const data = JSON.stringify([
    { name: '张三', age: 18 },
    { name: '李四', age: 20 }
  ])

  res.writeHead(200, {
    'Content-Type': 'application/json;charset=utf-8',
    'Access-Control-Allow-Origin': '*' // 允许所有来源跨域
  })
  res.end(data)
})

server.listen(3000, () => {
  console.log('CORS 服务已启动')
})
```

### 反向代理 —— 请求后端服务器

```js
const http = require('node:http')
const https = require('node:https')

// 获取第三方数据的通用 GET 函数
function httpGet(url) {
  return new Promise((resolve, reject) => {
    const client = url.startsWith('https') ? https : http
    client.get(url, (res) => {
      let data = ''
      res.on('data', (chunk) => { data += chunk })
      res.on('end', () => resolve(data))
    }).on('error', reject)
  })
}

const server = http.createServer(async (req, res) => {
  const urlObj = new URL(req.url, `http://${req.headers.host}`)

  res.setHeader('Content-Type', 'application/json;charset=utf-8')

  if (urlObj.pathname === '/api/proxy') {
    try {
      // 反向代理请求第三方 API
      const result = await httpGet(
        'https://jsonplaceholder.typicode.com/posts/1'
      )
      res.end(result)
    } catch (err) {
      res.statusCode = 502
      res.end(JSON.stringify({ error: '代理请求失败' }))
    }
  } else {
    res.statusCode = 404
    res.end(JSON.stringify({ error: 'Not Found' }))
  }
})

server.listen(3000, () => {
  console.log('代理服务已启动 → http://localhost:3000/api/proxy')
})
```

## 10. v24.x 原生服务器最佳实践

1. **使用 `node:` 前缀**：`require('node:http')`，清晰区分内置模块与第三方包。
2. **用 `URL` 类替代 `url.parse`**：`new URL(req.url, base)` 更现代、更安全。
3. **始终设置 `Content-Type` 响应头**：避免客户端猜测数据类型。
4. **必须调用 `res.end()`**：否则连接会一直挂起，最终导致客户端超时。
5. **始终处理错误**：监听 `req.on('error')` 和 `server.on('error')`。
6. **生产环境加反向代理**：Nginx / Caddy 放在 Node 前面处理静态资源、SSL、负载均衡。
7. **不要在生产环境直接用原生 http 构建大型应用**：原生模块适合学习原理和小工具；大型项目推荐 Express、Koa 或 Fastify 等框架。
