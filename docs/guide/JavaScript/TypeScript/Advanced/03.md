# 工具类型（Utility Types）

## 1. 概述

TypeScript 内置了大量**工具类型**（Utility Types），它们是基于泛型的类型变换器，可以用已有的类型生成新类型，大幅减少重复的类型定义。

> 以下按照使用频率和重要性排序。

## 2. 属性修饰类

### Partial\<T\> —— 全部变为可选

```typescript
interface User {
  name: string
  age: number
  email: string
}

// 所有属性变为可选
type PartialUser = Partial<User>
// { name?: string; age?: number; email?: string }

// 常用场景：更新函数
function updateUser(id: number, changes: Partial<User>) {
  // changes 只需要传需要更新的字段
}

updateUser(1, { age: 19 })               // ✅
updateUser(1, { name: 'newName', age: 20 }) // ✅
```

### Required\<T\> —— 全部变为必填

```typescript
interface Config {
  host?: string
  port?: number
  debug?: boolean
}

type FullConfig = Required<Config>
// { host: string; port: number; debug: boolean }

const config: FullConfig = {
  host: 'localhost',
  port: 3000,
  debug: true
  // 三个都不能少
}
```

### Readonly\<T\> —— 全部变为只读

```typescript
interface Todo {
  title: string
  completed: boolean
}

const todo: Readonly<Todo> = {
  title: '学习 TypeScript',
  completed: false
}

// todo.title = ''  // ❌ 只读属性不能修改
```

## 3. 选取与排除类

### Pick\<T, K\> —— 选取指定属性

```typescript
interface User {
  id: number
  name: string
  age: number
  email: string
  password: string
}

// 只取 id 和 name
type UserPreview = Pick<User, 'id' | 'name'>
// { id: number; name: string }

// 常用场景：API 返回的简化用户信息
const preview: UserPreview = { id: 1, name: 'yzb' }
```

### Omit\<T, K\> —— 排除指定属性

```typescript
// 排除 password
type UserWithoutPassword = Omit<User, 'password'>
// { id: number; name: string; age: number; email: string }

// 排除多个属性
type PublicUser = Omit<User, 'password' | 'email'>
// { id: number; name: string; age: number }

// 常用场景：序列化时去掉敏感字段
function toSafeUser(user: User): Omit<User, 'password'> {
  const { password, ...safe } = user
  return safe
}
```

## 4. 记录与映射类

### Record\<K, V\> —— 创建键值对类型

```typescript
// 创建一个以字符串为键、数字为值的映射
type PageViews = Record<string, number>

const views: PageViews = {
  '/home': 1200,
  '/about': 300,
  '/contact': 150
}

// 限定键的范围
type Role = 'admin' | 'user' | 'guest'
type Permissions = Record<Role, string[]>

const perms: Permissions = {
  admin: ['read', 'write', 'delete'],
  user: ['read', 'write'],
  guest: ['read']
}
```

### Exclude\<T, U\> —— 从联合类型中排除

```typescript
type AllColors = 'red' | 'green' | 'blue' | 'yellow'
type PrimaryColors = Exclude<AllColors, 'yellow'>
// 'red' | 'green' | 'blue'

type Numbers = 1 | 2 | 3 | 4 | 5
type Odd = Exclude<Numbers, 2 | 4>
// 1 | 3 | 5

// 排除 null 和 undefined
type T1 = Exclude<string | number | null | undefined, null | undefined>
// string | number
```

### Extract\<T, U\> —— 从联合类型中提取

```typescript
type AllShapes = 'circle' | 'square' | string
type SpecificShapes = Extract<AllShapes, 'circle' | 'square'>
// 'circle' | 'square'

// 提取函数类型
type Mixed = string | number | (() => void) | (() => string)
type Functions = Extract<Mixed, Function>
// (() => void) | (() => string)
```

## 5. 提取类型内部结构类

### ReturnType\<T\> —— 获取函数返回值类型

```typescript
function fetchUser(): Promise<{ id: number; name: string }> {
  return Promise.resolve({ id: 1, name: 'yzb' })
}

type UserResult = ReturnType<typeof fetchUser>
// Promise<{ id: number; name: string }>

// 常用在获取第三方函数返回值类型时
type ArrayPushResult = ReturnType<typeof Array.prototype.push>
// number
```

### Parameters\<T\> —— 获取函数参数类型（元组）

```typescript
function createUser(name: string, age: number, email?: string) {
  return { name, age, email }
}

type CreateUserParams = Parameters<typeof createUser>
// [name: string, age: number, email?: string]

// 可以解构使用
const params: CreateUserParams = ['yzb', 18]
```

### InstanceType\<T\> —— 获取构造函数的实例类型

```typescript
class User {
  name: string
  constructor(name: string) {
    this.name = name
  }
}

type UserInstance = InstanceType<typeof User>
// User

// 泛型工厂函数的典型用法
function createInstance<T extends new (...args: any[]) => any>(
  ctor: T,
  ...args: ConstructorParameters<T>
): InstanceType<T> {
  return new ctor(...args)
}

const user = createInstance(User, 'yzb')
// user 的类型为 User
```

### ConstructorParameters\<T\> —— 获取构造函数的参数类型

```typescript
class Rectangle {
  constructor(public width: number, public height: number) {}
}

type RectParams = ConstructorParameters<typeof Rectangle>
// [width: number, height: number]
```

## 6. 字符串操作类

```typescript
type Event = 'click' | 'change' | 'submit'

// 首字母大写
type CapitalizedEvent = Capitalize<Event>
// 'Click' | 'Change' | 'Submit'

// 首字母小写
type Uncapitalized = Uncapitalize<'HELLO'>
// 'hELLO'

// 全大写
type UppercaseEvent = Uppercase<Event>
// 'CLICK' | 'CHANGE' | 'SUBMIT'

// 全小写
type LowercaseEvent = Lowercase<UppercaseEvent>
// 'click' | 'change' | 'submit'
```

## 7. 非空处理类

### NonNullable\<T\> —— 排除 null 和 undefined

```typescript
type MaybeString = string | null | undefined
type DefiniteString = NonNullable<MaybeString>
// string

// 场景：数组过滤后去除 null
const items: (string | null)[] = ['a', null, 'b', null]
const nonNull = items.filter((item): item is string => item !== null)
// nonNull 类型为 string[]
```

## 8. 函数操作类

### Awaited\<T\> —— 获取 Promise resolve 的类型

```typescript
async function getData(): Promise<{ name: string; age: number }> {
  return { name: 'yzb', age: 18 }
}

type Data = Awaited<ReturnType<typeof getData>>
// { name: string; age: number }

// 支持递归解包嵌套 Promise
type Nested = Promise<Promise<number>>
type Unwrapped = Awaited<Nested>
// number
```

### ThisParameterType\<T\> 和 OmitThisParameter\<T\>

```typescript
function greet(this: { name: string }, greeting: string) {
  return `${greeting}, ${this.name}!`
}

type ThisType = ThisParameterType<typeof greet>
// { name: string }

type WithoutThis = OmitThisParameter<typeof greet>
// (greeting: string) => string
```

## 9. 类型断言工具函数

虽然不是工具类型，但以下函数配合工具类型非常实用：

```typescript
// 确保 switch 穷举所有情况
function assertNever(value: never): never {
  throw new Error(`未处理的值: ${value}`)
}

type Shape =
  | { kind: 'circle'; radius: number }
  | { kind: 'square'; side: number }
  | { kind: 'triangle'; base: number; height: number }

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':
      return Math.PI * shape.radius ** 2
    case 'square':
      return shape.side ** 2
    case 'triangle':
      return (shape.base * shape.height) / 2
    default:
      // 如果漏写了 case，这里会报编译错误
      return assertNever(shape)
  }
}
```

## 10. 工具类型组合实战

### 案例一：表单类型

```typescript
interface UserForm {
  username: string
  password: string
  confirmPassword: string
  email: string
  remember: boolean
}

// 登录只需要 username 和 password
type LoginForm = Pick<UserForm, 'username' | 'password'>

// 提交时不需要 confirmPassword（前后端校验）
type SubmitForm = Omit<UserForm, 'confirmPassword'>

// 编辑时所有字段可选
type EditForm = Partial<UserForm>

// 只读展示用
type ReadonlyForm = Readonly<UserForm>
```

### 案例二：API 响应类型

```typescript
// 基础响应
interface ApiResponse<T> {
  code: number
  message: string
  data: T
}

// 分页数据
interface PageData<T> {
  list: T[]
  total: number
  page: number
  pageSize: number
}

// 用户详情
interface User {
  id: number
  name: string
  email: string
  createdAt: Date
  updatedAt: Date
}

// 组合得到具体接口类型
type UserListResponse = ApiResponse<PageData<Omit<User, 'email'>>>
// data.list 中不包含 email 字段

type UserDetailResponse = ApiResponse<User>

// 导出给前端使用的 Promise 类型
type FetchUserList = () => Promise<UserListResponse>
type FetchUserDetail = (id: number) => Promise<UserDetailResponse>
```

### 案例三：权限系统

```typescript
type Permission = 'read' | 'write' | 'delete' | 'admin'

// 角色
type Role = 'guest' | 'user' | 'manager' | 'admin'

// 每种角色的权限
type RolePermissions = Record<Role, Permission[]>

const permissions: RolePermissions = {
  guest: ['read'],
  user: ['read', 'write'],
  manager: ['read', 'write', 'delete'],
  admin: ['read', 'write', 'delete', 'admin']
}

// 检查是否有某个权限
function hasPermission(
  role: Role,
  permission: Permission
): boolean {
  return permissions[role].includes(permission)
}
```

## 11. 工具类型速查表

| 工具类型 | 功能 | 示例 |
|----------|------|------|
| `Partial<T>` | 全部变为可选 | `Partial<User>` |
| `Required<T>` | 全部变为必填 | `Required<Config>` |
| `Readonly<T>` | 全部变为只读 | `Readonly<Todo>` |
| `Pick<T, K>` | 选取指定属性 | `Pick<User, 'id' \| 'name'>` |
| `Omit<T, K>` | 排除指定属性 | `Omit<User, 'password'>` |
| `Record<K, V>` | 创建键值对映射 | `Record<string, number>` |
| `Exclude<T, U>` | 从联合中排除 | `Exclude<Color, 'red'>` |
| `Extract<T, U>` | 从联合中提取 | `Extract<T, Function>` |
| `NonNullable<T>` | 排除 null/undefined | `NonNullable<string \| null>` |
| `ReturnType<T>` | 获取函数返回值 | `ReturnType<typeof fn>` |
| `Parameters<T>` | 获取函数参数元组 | `Parameters<typeof fn>` |
| `InstanceType<T>` | 获取类实例类型 | `InstanceType<typeof User>` |
| `ConstructorParameters<T>` | 获取构造参数 | `ConstructorParameters<typeof C>` |
| `Awaited<T>` | 获取 Promise 解包类型 | `Awaited<Promise<number>>` |
| `Capitalize<T>` | 首字母大写 | `Capitalize<'hello'>` |
| `Uncapitalize<T>` | 首字母小写 | `Uncapitalize<'Hello'>` |
| `Uppercase<T>` | 全大写 | `Uppercase<'hello'>` |
| `Lowercase<T>` | 全小写 | `Lowercase<'HELLO'>` |

## 12. 工具类型最佳实践

1. **组合使用**：单个工具类型能力有限，组合起来非常强大（如 `Partial<Omit<User, 'id'>>`）。
2. **善用 `Pick` 和 `Omit`**：它们是最常用的工具类型，用于从大接口中衍生小接口。
3. **遇到 `any` 时用 `NonNullable`**：`NonNullable<T>` 比 `T & {}` 更安全。
4. **`Awaited` 处理嵌套 Promise**：`Awaited` 递归解包比手动 `infer` 更方便。
5. **字符串工具类型**配合模板字面量类型生成更强的类型约束。
