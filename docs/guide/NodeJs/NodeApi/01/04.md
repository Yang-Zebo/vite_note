# Node.js 文件操作

## 1. fs 模块概述

1. `fs` 是 `File System`（文件系统）的缩写，是 Node.js 内置的核心模块。
2. 通过 `fs` 模块可以在任何操作系统（Windows、macOS、Linux）上操作文件和目录。
3. `fs` 模块提供了三种风格的 API：
   - **Promise API**（推荐，v10+ 稳定）：异步、非阻塞，返回 Promise。
   - **Callback API**（传统）：异步、非阻塞，通过回调函数传递结果。
   - **Synchronous API**（同步）：阻塞当前线程，直到操作完成。

```js
// ESM 导入方式（v24.x 推荐，使用 node: 前缀）
import fs from 'node:fs'
import fsp from 'node:fs/promises'

// CJS 导入方式（传统）
const fs = require('node:fs')
const fsp = require('node:fs/promises')
```

4. 在 v24.x 中，**推荐使用 `node:fs/promises`** 配合 `async/await` 进行所有文件操作，代码更清晰、更容易维护。

## 2. 三种 API 风格对比

```js
import { readFile } from 'node:fs/promises'
import fs from 'node:fs'

// 1. Promise API —— 推荐
try {
  const data = await readFile('./test.txt', 'utf-8')
  console.log(data)
} catch (err) {
  console.error('读取失败:', err.message)
}

// 2. Callback API —— 传统写法
fs.readFile('./test.txt', 'utf-8', (err, data) => {
  if (err) {
    console.error('读取失败:', err.message)
    return
  }
  console.log(data)
})

// 3. Synchronous API —— 同步阻塞，仅在脚本初始化时使用
try {
  const data = fs.readFileSync('./test.txt', 'utf-8')
  console.log(data)
} catch (err) {
  console.error('读取失败:', err.message)
}
```

| 风格 | 阻塞 | 适用场景 |
|------|------|----------|
| Promise (`fs/promises`) | 否 | **绝大多数场景**，配合 `async/await` 使用 |
| Callback (`fs`) | 否 | 老旧代码兼容，不推荐新项目使用 |
| Sync (`fs.xxxSync`) | 是 | 仅限脚本启动初始化、CLI 工具等简单场景 |

## 3. 文件路径

### 路径类型

1. `fs` 模块接受三种路径类型：`string`、`Buffer`、`URL`。

```js
import { readFile } from 'node:fs/promises'

// 字符串路径
await readFile('./test.txt')

// Buffer 路径
await readFile(Buffer.from('./test.txt'))

// URL 路径（v24.x 推荐用于 ESM 项目）
const fileUrl = new URL('./test.txt', import.meta.url)
await readFile(fileUrl)
```

2. 在 ESM 中没有 `__dirname` 和 `__filename`，需要用 `import.meta.url` 替代：

```js
import { fileURLToPath } from 'node:url'
import { dirname } from 'node:path'

const __filename = fileURLToPath(import.meta.url)
const __dirname = dirname(__filename)
```

### 路径安全检查

```js
import { resolve, normalize } from 'node:path'

// 始终使用 resolve 获取绝对路径，避免相对路径带来的不确定性
const safePath = resolve('./data/user.txt')
console.log(safePath) // /Users/yzb/project/data/user.txt

// normalize 用于清理路径中的冗余部分
console.log(normalize('/a//b/../c/./d')) // '/a/c/d'
```

## 4. 读取文件

### 一次性读取全部内容

```js
import { readFile } from 'node:fs/promises'

// 默认返回 Buffer
const buffer = await readFile('./test.txt')
console.log(buffer) // <Buffer 68 65 6c 6c 6f>

// 指定编码，返回字符串
const text = await readFile('./test.txt', 'utf-8')
console.log(text) // 'hello world'

// 不指定编码时手动转换
const buf = await readFile('./test.txt')
console.log(buf.toString('utf-8'))
```

> **注意**：`readFile` 会将整个文件内容加载到内存中。对于大文件（如几百 MB 以上），应使用流式读取。

### 流式读取（大文件场景）

对超大文件（如日志文件、视频），使用 `createReadStream` 分段读取，避免一次加载全部内容到内存：

```js
import { createReadStream } from 'node:fs'

const rs = createReadStream('./large-file.log', {
  encoding: 'utf-8',
  highWaterMark: 64 * 1024 // 每次读取 64KB
})

rs.on('data', (chunk) => {
  console.log('收到数据块，大小:', chunk.length)
  // 逐块处理数据
})

rs.on('end', () => {
  console.log('文件读取完毕')
})

rs.on('error', (err) => {
  console.error('读取出错:', err.message)
})
```

### 逐行读取

```js
import { createReadStream } from 'node:fs'
import { createInterface } from 'node:readline'

const rl = createInterface({
  input: createReadStream('./access.log'),
  crlfDelay: Infinity // 处理 Windows/Mac/Linux 的换行符差异
})

// 监听每一行
rl.on('line', (line) => {
  console.log('行内容:', line)
})

rl.on('close', () => {
  console.log('文件读取完毕')
})
```

## 5. 写入文件

### 一次性写入

```js
import { writeFile } from 'node:fs/promises'

// 写入字符串（默认覆盖已有文件）
await writeFile('./output.txt', 'Hello Node.js!')

// 指定编码
await writeFile('./output.txt', '你好，世界', 'utf-8')
```

### 追加内容

`appendFile` 在文件末尾追加内容，文件不存在则自动创建：

```js
import { appendFile } from 'node:fs/promises'

await appendFile('./log.txt', `[${new Date().toISOString()}] 用户登录\n`)
await appendFile('./log.txt', `[${new Date().toISOString()}] 数据查询\n`)

// log.txt 内容会被逐行追加
```

### 流式写入

```js
import { createWriteStream } from 'node:fs'

const ws = createWriteStream('./output.txt', { flags: 'a' }) // 'a' 表示追加模式

ws.write('第一行\n')
ws.write('第二行\n')
ws.write('第三行\n')

ws.end() // 必须调用 end() 确保数据写入完毕

// 监听完成事件
ws.on('finish', () => {
  console.log('写入完成')
})
```

### 管道操作（读 → 写）

```js
import { createReadStream, createWriteStream } from 'node:fs'

const rs = createReadStream('./source.txt')
const ws = createWriteStream('./copy.txt')

rs.pipe(ws)

ws.on('finish', () => {
  console.log('文件复制完成')
})
```

## 6. 文件元数据

### stat —— 获取文件信息

```js
import { stat } from 'node:fs/promises'

const stats = await stat('./test.txt')

console.log('文件大小:', stats.size, '字节')
console.log('是否为文件:', stats.isFile())
console.log('是否为目录:', stats.isDirectory())
console.log('是否为符号链接:', stats.isSymbolicLink())
console.log('创建时间:', stats.birthtime)
console.log('修改时间:', stats.mtime)
console.log('访问时间:', stats.atime)
```

### access —— 检查文件权限

`access` 用于检查当前进程对文件的访问权限，不会读取文件内容：

```js
import { access, constants } from 'node:fs/promises'

try {
  await access('./secret.key', constants.R_OK | constants.W_OK)
  console.log('文件可读可写')
} catch {
  console.log('没有读写权限或文件不存在')
}

// constants 常用值：
// F_OK  —— 文件是否存在
// R_OK  —— 是否可读
// W_OK  —— 是否可写
// X_OK  —— 是否可执行
```

### chmod —— 修改文件权限

```js
import { chmod } from 'node:fs/promises'

// 设置为所有者可读写，组和其他人只读
await chmod('./script.sh', 0o644)

// 设置为所有者可读写执行（常用于脚本文件）
await chmod('./run.sh', 0o755)
```

### utimes —— 修改文件时间戳

```js
import { utimes } from 'node:fs/promises'

const now = new Date()
const yesterday = new Date(Date.now() - 86400000)

// 修改文件的访问时间和修改时间
await utimes('./test.txt', now, yesterday)
// 参数: (路径, 访问时间, 修改时间)
```

## 7. 文件标志（Flags）

通过 `flags` 控制在打开文件时的行为：

| Flag | 含义 |
|------|------|
| `'r'` | 只读，文件必须存在（默认值） |
| `'r+'` | 读写，文件必须存在 |
| `'w'` | 只写，不存在则创建，存在则清空 |
| `'w+'` | 读写，不存在则创建，存在则清空 |
| `'a'` | 追加写入，不存在则创建 |
| `'a+'` | 追加读写，不存在则创建 |
| `'x'` | 只写，创建文件，如果已存在则失败 |
| `'x+'` | 读写，创建文件，如果已存在则失败 |

```js
import { writeFile } from 'node:fs/promises'

// 使用 'x' flag 防止意外覆盖已有文件
try {
  await writeFile('./config.json', data, { flag: 'wx' })
} catch (err) {
  if (err.code === 'EEXIST') {
    console.log('配置文件已存在，拒绝覆盖')
  }
}
```

## 8. 目录操作

### 创建目录

```js
import { mkdir } from 'node:fs/promises'

// 创建单层目录
await mkdir('./new-dir')

// 递归创建多级目录（类似于 mkdir -p）
await mkdir('./a/b/c/d', { recursive: true })
```

### 读取目录内容

```js
import { readdir } from 'node:fs/promises'

// 基础用法：返回文件名数组
const files = await readdir('./project')
console.log(files) // ['index.js', 'utils.js', 'package.json']

// 获取详细信息
const entries = await readdir('./project', { withFileTypes: true })
for (const entry of entries) {
  if (entry.isDirectory()) {
    console.log(`📁 ${entry.name}`)
  } else if (entry.isFile()) {
    console.log(`📄 ${entry.name}`)
  }
}
```

### 删除目录

```js
import { rm } from 'node:fs/promises'

// 删除空目录
await rm('./empty-dir', { recursive: false })

// 递归删除整个目录及其内容（类似于 rm -rf）
await rm('./old-project', { recursive: true, force: true })

// 注意：rm 是 v14.14+ 引入的，已替代旧版 rmdir
// rmdir 只能删除空目录，不推荐继续使用
```

### 遍历目录（递归获取所有文件）

```js
import { readdir } from 'node:fs/promises'
import { resolve } from 'node:path'

async function walkDir(dirPath) {
  const result = []
  const entries = await readdir(dirPath, { withFileTypes: true })
  
  for (const entry of entries) {
    const fullPath = resolve(dirPath, entry.name)
    if (entry.isDirectory()) {
      // 递归遍历子目录
      const subFiles = await walkDir(fullPath)
      result.push(...subFiles)
    } else {
      result.push(fullPath)
    }
  }
  
  return result
}

const allFiles = await walkDir('./project')
console.log(allFiles)
```

## 9. 文件操作

### 复制文件

```js
import { copyFile } from 'node:fs/promises'

await copyFile('./source.txt', './dest.txt')

// copyFile 的第三个参数决定复制行为
import { constants } from 'node:fs'
// constants.COPYFILE_EXCL —— 目标文件已存在则失败
// constants.COPYFILE_FICLONE —— 使用写时复制（copy-on-write）
await copyFile('./source.txt', './dest.txt', constants.COPYFILE_EXCL)
```

### 移动 / 重命名文件

```js
import { rename } from 'node:fs/promises'

// 重命名
await rename('./old-name.txt', './new-name.txt')

// 移动（路径变化就是移动）
await rename('./temp/data.txt', './archive/data.txt')
```

### 删除文件

```js
import { unlink, rm } from 'node:fs/promises'

// 删除单个文件
await unlink('./obsolete.txt')

// rm 也能删除文件（推荐统一使用 rm）
await rm('./obsolete.txt')
// 配合 force: true 忽略文件不存在的情况
await rm('./obsolete.txt', { force: true })
```

### 截断文件

`truncate` 用于将文件截断到指定长度（默认 0，即清空文件）：

```js
import { truncate } from 'node:fs/promises'

// 清空文件内容
await truncate('./log.txt')

// 截断到指定字节
await truncate('./data.bin', 1024) // 只保留前 1024 字节
```

## 10. 文件描述符（低级操作）

对于需要精细控制的场景，可以直接使用文件描述符：

```js
import { open, read, write, close } from 'node:fs/promises'

// 打开文件
const fd = await open('./data.txt', 'a+')

// 写入数据
const writeResult = await fd.write('追加的内容\n')
console.log('写入了', writeResult.bytesWritten, '字节')

// 读取指定位置的数据
const buffer = Buffer.alloc(100)
const readResult = await fd.read(buffer, 0, 100, 0)
console.log('读取了', readResult.bytesRead, '字节')
console.log(buffer.subarray(0, readResult.bytesRead).toString())

// 手动关闭文件描述符
await fd.close()
```

> 使用 `open()` 获取文件句柄的好处：可以多次读/写而无需每次重新打开文件，适合高频操作场景。

## 11. 文件监听

### watch —— 监听文件或目录变化

```js
import { watch } from 'node:fs'

const watcher = watch('./project', { recursive: true }, (eventType, filename) => {
  console.log(`检测到变化: ${eventType} — ${filename}`)
  // eventType: 'rename'（创建/删除/重命名）或 'change'（修改）
})

// 停止监听
// watcher.close()
```

### watchFile —— 轮询方式监听

每隔一段时间检查文件，比 `watch` 更稳定但性能开销更大：

```js
import { watchFile, unwatchFile } from 'node:fs'

watchFile('./config.json', { interval: 1000 }, (curr, prev) => {
  console.log('文件修改时间变化:')
  console.log('  之前:', prev.mtime)
  console.log('  现在:', curr.mtime)
})

// 停止监听
// unwatchFile('./config.json')
```

| 方法 | 原理 | 性能 | 可靠性 |
|------|------|------|--------|
| `watch` | 操作系统原生事件（inotify / FSEvents / ReadDirectoryChangesW） | 高 | 在不同平台上有细微差异 |
| `watchFile` | 轮询 stat | 低 | 跨平台行为一致 |

## 12. 临时文件

Node.js 提供了 `fs.mkdtemp` 用于创建临时目录：

```js
import { mkdtemp } from 'node:fs/promises'
import { tmpdir } from 'node:os'
import { join } from 'node:path'

// 在系统临时目录下创建临时文件夹
const tmpDir = await mkdtemp(join(tmpdir(), 'my-app-'))
console.log('临时目录:', tmpDir)
// 输出类似: /tmp/my-app-xxxxxx

// 在临时目录中操作...
// 使用完毕后记得清理
import { rm } from 'node:fs/promises'
await rm(tmpDir, { recursive: true, force: true })
```

## 13. 错误处理最佳实践

```js
import { readFile, writeFile, access } from 'node:fs/promises'

async function safeReadConfig(configPath) {
  // 1. 先检查文件是否存在
  try {
    await access(configPath)
  } catch {
    console.log('配置文件不存在，使用默认配置')
    return { host: 'localhost', port: 3000 }
  }
  
  // 2. 读取并解析
  try {
    const content = await readFile(configPath, 'utf-8')
    return JSON.parse(content)
  } catch (err) {
    if (err.name === 'SyntaxError') {
      console.error('配置文件格式错误:', err.message)
    } else {
      console.error('读取配置文件失败:', err.message)
    }
    // 出错时返回默认配置
    return { host: 'localhost', port: 3000 }
  }
}

// 3. 写入文件也需要错误处理
async function safeWriteConfig(configPath, config) {
  try {
    const data = JSON.stringify(config, null, 2)
    await writeFile(configPath, data, 'utf-8')
    console.log('配置已保存')
  } catch (err) {
    console.error('写入配置文件失败:', err.message)
  }
}
```

### 常见错误码

| 错误码 | 含义 |
|--------|------|
| `ENOENT` | 文件或目录不存在 |
| `EACCES` | 权限不足 |
| `EEXIST` | 文件已存在（配合 `x` flag 时） |
| `EISDIR` | 期望文件但路径是目录 |
| `ENOTDIR` | 期望目录但路径是文件 |
| `ENOTEMPTY` | 目录非空，无法删除 |
| `EMFILE` | 打开的文件描述符过多 |

错误码判断示例：

```js
import { unlink } from 'node:fs/promises'

try {
  await unlink('./maybe-not-exist.txt')
} catch (err) {
  if (err.code === 'ENOENT') {
    console.log('文件本来就不存在，无需删除')
  } else {
    throw err // 其它错误继续向上抛出
  }
}
```

## 14. 综合实战案例

### 案例一：配置文件读写

```js
import { readFile, writeFile, access } from 'node:fs/promises'
import { resolve, dirname } from 'node:path'
import { fileURLToPath } from 'node:url'

const __dirname = dirname(fileURLToPath(import.meta.url))
const configPath = resolve(__dirname, 'app-config.json')

const defaultConfig = {
  server: { host: '0.0.0.0', port: 3000 },
  database: { url: 'mongodb://localhost:27017/myapp' }
}

async function loadConfig() {
  try {
    await access(configPath)
    const raw = await readFile(configPath, 'utf-8')
    return { ...defaultConfig, ...JSON.parse(raw) }
  } catch {
    await writeFile(configPath, JSON.stringify(defaultConfig, null, 2))
    return defaultConfig
  }
}

const config = await loadConfig()
console.log('启动服务器:', config.server)
```

### 案例二：日志轮转

当日志文件超过指定大小时，自动归档并创建新日志：

```js
import { stat, rename, appendFile } from 'node:fs/promises'

const MAX_LOG_SIZE = 5 * 1024 * 1024 // 5MB
const LOG_PATH = './app.log'

async function writeLog(message) {
  try {
    const stats = await stat(LOG_PATH)
    if (stats.size > MAX_LOG_SIZE) {
      const timestamp = new Date().toISOString().replace(/[:.]/g, '-')
      await rename(LOG_PATH, `./app-${timestamp}.log`)
      console.log('日志已归档')
    }
  } catch {
    // 文件不存在，无需轮转，直接写入
  }
  
  const line = `[${new Date().toISOString()}] ${message}\n`
  await appendFile(LOG_PATH, line)
}

await writeLog('用户 admin 登录成功')
await writeLog('数据备份完成')
```

### 案例三：项目脚手架 —— 从模板复制文件结构

```js
import { mkdir, readFile, writeFile, readdir } from 'node:fs/promises'
import { resolve, join, extname } from 'node:path'

async function scaffoldProject(targetDir, templateDir, variables) {
  // 确保目标目录存在
  await mkdir(targetDir, { recursive: true })
  
  const entries = await readdir(templateDir, { withFileTypes: true })
  
  for (const entry of entries) {
    const srcPath = join(templateDir, entry.name)
    const destPath = join(targetDir, entry.name)
    
    if (entry.isDirectory()) {
      await mkdir(destPath, { recursive: true })
      await scaffoldProject(destPath, srcPath, variables)
    } else {
      let content = await readFile(srcPath, 'utf-8')
      
      // 替换模板变量 {{key}} → 实际值
      for (const [key, value] of Object.entries(variables)) {
        content = content.replaceAll(`{{${key}}}`, value)
      }
      
      await writeFile(destPath, content)
      console.log(`  已生成: ${destPath}`)
    }
  }
}

// 使用示例
await scaffoldProject('./my-new-app', './template', {
  name: 'my-new-app',
  author: 'yzb',
  version: '1.0.0'
})
```

## 15. fs 常用 API 速查表

| 操作 | Promise API | 说明 |
|------|-------------|------|
| 读取文件 | `readFile(path, [encoding])` | 一次性读取全部内容 |
| 写入文件 | `writeFile(path, data, [encoding])` | 覆盖写入 |
| 追加写入 | `appendFile(path, data, [encoding])` | 在末尾追加 |
| 复制文件 | `copyFile(src, dest, [flags])` | 复制文件 |
| 删除文件 | `unlink(path)` / `rm(path)` | 删除文件 |
| 重命名 | `rename(old, new)` | 重命名或移动 |
| 文件信息 | `stat(path)` | 获取元数据 |
| 权限检查 | `access(path, [mode])` | 检查访问权限 |
| 修改权限 | `chmod(path, mode)` | 修改文件权限 |
| 修改时间 | `utimes(path, atime, mtime)` | 修改时间戳 |
| 截断文件 | `truncate(path, [len])` | 截断到指定长度 |
| 创建目录 | `mkdir(path, [options])` | 创建目录 |
| 读取目录 | `readdir(path, [options])` | 列出目录内容 |
| 删除目录 | `rm(path, [options])` | 递归删除 |
| 打开文件 | `open(path, [flags])` | 返回 FileHandle |
| 监听变化 | `watch(path, [options], callback)` | 返回 watcher 对象 |

## 16. Node.js v24.x 文件操作最佳实践

1. **优先使用 `node:fs/promises`**，配合 `async/await`，代码可读性最好。
2. **使用 `node:` 前缀**引用核心模块（`import fs from 'node:fs'`），明确区分内置模块和第三方包。
3. **始终处理错误**：所有文件操作都可能失败（路径不存在、权限不足、磁盘满等），务必 `try/catch`。
4. **大文件用流**：对于 > 100MB 的文件，使用 `createReadStream` / `createWriteStream` 避免 OOM。
5. **避免同步 API** 在请求处理中使用：`xxxSync` 会阻塞事件循环，仅在脚本启动阶段使用。
6. **用 `rm` 替代 `rmdir`**：`rmdir` 只能删除空目录，`rm` 支持递归删除（v14.14+）。
7. **路径始终用绝对路径**：用 `resolve()` 拼接，避免相对路径带来的意外。
8. **`force: true` 防报错**：删除文件时设置 `{ force: true }` 避免因文件不存在而抛出异常。
9. **使用 flag `'x'` 防止误覆盖**：创建文件时用 `writeFile(path, data, { flag: 'wx' })`，如果文件已存在会抛出 `EEXIST` 错误。
10. **`fs/promises` 也导出了 `constants`**：`import { constants } from 'node:fs/promises'` 在 v24.x 中可用，但 `import { constants } from 'node:fs'` 更通用。
