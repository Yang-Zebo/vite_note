# Node 常用模块

## 1. path 模块

### 概述

1. `path` 模块用于对路径和文件进行处理，提供了很多好用的方法。
2. 在`macOS`、`Linux`和`Windows`上的路径是不一样的。
    - `Windows`上会使用 `\`或者 `\\` 来作为文件路径的分隔符，当然目前也支持 `/`。
    - 在`macOS`、`Linux`的`Unix`操作系统上使用 `/` 来作为文件路径的分隔符。
3. 如果在 `Windows` 上使用 `\` 来作为分隔符开发了一个应用程序，要部署到 `Linux`
   上，显示路径会出现一些问题，所以为了屏蔽它们之间的差异，在开发中对于路径的操作我们可以使用 `path` 模块。

### path 常见 api

> **v24.x 推荐使用 `node:` 前缀引入**：`import path from 'node:path'` 或 `const path = require('node:path')`

- `dirname(path)`，获取文件所在的文件夹。

```js
const path = require('path')
path.dirname('./minivue/koa.js') // './minivue'
```

- `basename(path)`，获取文件名。

```js
const path = require('path')
path.basename('./minivue/koa.js') // 'koa.js'
```

- `extname(path)`，获取文件的扩展名（文件格式）。

```js
const path = require('path')
path.extname('./minivue/koa.js') // '.js'
```

`join`路径拼接。如果希望将多个路径进行拼接，但是不同的操作系统可能使用的是不同的分隔符，这个时候我们可以使用`join`函数。

- `join`会智能拼接 `./`，`../`，`//`，`/`，`\` 等路径，并且根据系统转为系统需要的路径。

```js
const path = require('path')
path.join('a', 'b', 'c') // 'a/b/c'
path.join('/a', 'b', 'c') // '/a/b/c'
path.join('/a', '\\b', '/c') // '/a/b/c'
```

- `resolve`，一般用于将文件和某个文件夹拼接。
    - 不带参数时。`resolve()` 返回的是当前的文件所在文件夹的绝对路径。作用与`__dirname`相同。
    - 已`/`开头的参数，会变成参数的拼接，也会智能化转换路径。（会以最后一个`/`开头的参数开始）

```js
path.resolve('/a', 'b') // '/a/b'
```

- 带不是`/`开头的参数时（一般只看第一个参数），返回的是当前绝对路径拼接参数路径。

```js
path.resolve('a') // /xxx/xxx/xxx/a
path.resolve('a', 'b') // /xxx/xxx/xxx/a/b

path.resolve('./a') // /xxx/xxx/xxx/a
path.resolve('./a', './b') // /xxx/xxx/xxx/a/b
```

## 2. fs 模块

### 概述

1. `fs` 是 `File System` 的缩写，表示文件系统，`fs` 模块是 Node.js 的文件系统。
2. 借助于 Node 帮我们封装的文件系统，我们可以在任何操作系统（Windows、macOS、Linux）上面直接去操作文件。
3. Node 文件系统的 API 非常多，日常使用中只需要学习最常用的即可，更多的应该是等用到的时候查询即可。
4. **v24.x 推荐使用 `node:fs/promises` + `async/await`** 进行所有文件操作。更多文件操作技巧，请参考 [文件操作笔记](/guide/NodeJs/NodeApi/01/04)。

### 导入方式

```js
// ESM（推荐）
import fs from 'node:fs'
import fsp from 'node:fs/promises'

// CJS（传统）
const fs = require('node:fs')
const fsp = require('node:fs/promises')
```

### 三种操作方式

1. **Promise API（推荐）**：返回 Promise，配合 `async/await` 使用，代码不会阻塞。

```js
import { readFile } from 'node:fs/promises'

const data = await readFile('./test.txt', 'utf-8')
console.log(data)
```

2. **Callback API（传统）**：通过回调函数传递结果，代码不会阻塞。第一个参数为错误，第二个参数为结果。

```js
import fs from 'node:fs'

fs.readFile('./test.txt', 'utf-8', (err, data) => {
  if (err) {
    console.log(err)
    return
  }
  console.log(data)
})
```

3. **Sync API（同步）**：代码会被阻塞，直到操作完成。仅在脚本初始化时使用。

```js
import fs from 'node:fs'

const data = fs.readFileSync('./test.txt', 'utf-8')
console.log(data)
```

### 文件信息 stat

`stat` 用于获取文件或文件夹的元数据。

```js
import { stat } from 'node:fs/promises'

const stats = await stat('./test.txt')
console.log(stats.isFile())        // 是否为一个文件
console.log(stats.isDirectory())   // 是否为一个文件夹
console.log(stats.size)            // 文件大小（字节）
console.log(stats.birthtime)       // 创建时间
console.log(stats.mtime)           // 最后修改时间
```

### 文件夹操作

**新建文件夹**：`mkdir(path, { recursive: true })` 创建文件夹，`recursive: true` 可一次性创建多级目录。

```js
import { mkdir } from 'node:fs/promises'

await mkdir('./xx/xx', { recursive: true })
```

**读取文件夹内容**：`readdir(path, { withFileTypes: true })` 获取文件列表。

```js
import { readdir } from 'node:fs/promises'

// 返回文件名字符串数组
const files = await readdir('./xx')

// 返回带类型信息的数组
const entries = await readdir('./xx', { withFileTypes: true })
for (const entry of entries) {
  if (entry.isDirectory()) {
    console.log('文件夹:', entry.name)
  } else {
    console.log('文件:', entry.name)
  }
}
```

**重命名/移动**：`rename(oldPath, newPath)` 可重命名或移动文件/文件夹。

```js
import { rename } from 'node:fs/promises'

await rename('./files', './files-new')
```

**删除文件夹**：使用 `rm(path, { recursive: true })` 递归删除（v14.14+），旧版 `rmdir` 只能删除空目录，**不推荐继续使用**。

```js
import { rm } from 'node:fs/promises'

await rm('./files', { recursive: true, force: true })
// force: true 表示文件夹不存在时也不报错
```

### 文件操作

**写入文件**：`writeFile(path, data, [encoding])`，文件不存在则创建，存在则覆盖。

```js
import { writeFile } from 'node:fs/promises'

await writeFile('./files/1.txt', 'hello', 'utf-8')
```

**追加内容**：`appendFile`，文件不存在则创建，存在则在末尾追加。

```js
import { appendFile } from 'node:fs/promises'

await appendFile('./files/2.txt', '\nworld')
```

**读取文件**：`readFile(path, [encoding])`。不指定编码返回 Buffer，指定编码返回字符串。

```js
import { readFile } from 'node:fs/promises'

// Buffer
const buf = await readFile('./files/1.txt')
// 字符串
const text = await readFile('./files/1.txt', 'utf-8')
```

**删除文件**：`unlink(path)` 或 `rm(path)`。

```js
import { unlink, rm } from 'node:fs/promises'

// unlink 删除文件
await unlink('./files/2.txt')
// rm 也可删除文件（推荐统一使用 rm）
await rm('./files/2.txt', { force: true })
```

### 文件复制

**单个文件复制**：`copyFile(src, dest)`

```js
import { copyFile } from 'node:fs/promises'

await copyFile('./source.txt', './dest.txt')
```

**批量复制文件**（综合案例）：

```js
import { readdir, mkdir, copyFile } from 'node:fs/promises'
import { resolve, extname } from 'node:path'

const args = process.argv
let sourceDirName = args[2]
let targetDirName = args[3] || sourceDirName + ' copy'
let copyType = args[4]           // 可选：只复制指定后缀的文件

const currentDirSrc = resolve('')
const sourceDir = resolve(currentDirSrc, sourceDirName)
const targetDir = resolve(currentDirSrc, targetDirName)

async function ensureDir(dirPath) {
  try {
    await mkdir(dirPath, { recursive: true })
  } catch {
    console.log('文件夹已存在')
  }
}

async function batchCopyFiles(sourceDir, targetDir) {
  const entries = await readdir(sourceDir, { withFileTypes: true })
  for (const entry of entries) {
    const srcPath = resolve(sourceDir, entry.name)
    const destPath = resolve(targetDir, entry.name)
    if (entry.isDirectory()) {
      await ensureDir(destPath)
      await batchCopyFiles(srcPath, destPath)
    } else {
      if (!copyType || extname(entry.name) === '.' + copyType) {
        await copyFile(srcPath, destPath)
        console.log('复制成功:', entry.name)
      }
    }
  }
}

console.log('开始复制...')
await ensureDir(targetDir)
await batchCopyFiles(sourceDir, targetDir)
console.log('复制完成')
```

### 文件描述符

1. Node.js 抽象出操作系统之间的特定差异，并为所有打开的文件分配一个数字型的文件描述符。
2. `fs.open()` 用于打开文件，返回 `FileHandle` 对象（Promise API），可以多次读写而无需重复打开。

```js
import { open } from 'node:fs/promises'

// Promise API
const fd = await open('./test.txt', 'r')
console.log(fd.fd)  // 文件描述符数字
await fd.close()

// Callback API
import fs from 'node:fs'
fs.open('./test.txt', 'r', (err, fd) => {
  console.log(fd)
})
```

### 流式操作

当文件太大，需要分段读取、分段写入，避免一次性将全部内容加载到内存。

```js
import { createReadStream, createWriteStream } from 'node:fs'

const rs = createReadStream('./files/1.txt', 'utf-8')
const ws = createWriteStream('./files/3.txt', 'utf-8')

// 方式一：手动监听 data 事件
rs.on('data', (chunk) => {
  ws.write(chunk)
  console.log('写入数据块:', chunk.length, '字节')
})
rs.on('end', () => {
  console.log('读取完毕')
  ws.end()
})

// 方式二：使用 pipe 自动分段写入（更简洁）
rs.pipe(ws)
```

## 3. events 模块

### 概况

1. `Node` 中的核心 `API` 都是基于异步事件驱动的，对象（`Emitters`）发出某一个事件。
2. 监听这个事件（监听器 `Listeners`），并且传入的回调函数，这个回调函数会在监听到事件时调用。
3. 发出事件和监听事件都是通过 `EventEmitter` 类来完成的，它们都属于 `events` 模块。

```js
const EventEmitter = require('node:events')
const bus = new EventEmitter()
bus.on('play', (val) => {
  console.log('播放', val)
})
bus.emit('play', 'value')
```

### 常见方法

监听事件：`emitter.on(eventName, listener)` 或 `addListener`。

```js
function clickHandle(name) {
  console.log(name)
}

bus.on('click', clickHandle)
```

发出事件：`emitter.emit(eventName, ...args)`。

```js
// 可以携带参数
bus.emit('click', 'yzb')
// 打印 'yzb'
```

移除事件监听：`emitter.off(eventName, listener)` 或 `removeListener`。

```js
bus.off('click', clickHandle)
```

### 补充方法

`emitter.once(eventName, listener)`：事件监听一次。

```js
bus.once('click', args => {
  console.log('监听一次', args)
})
```

`emitter.prependListener(eventName, listener)`；将监听事件添加到最前面。
`emitter.prependOnceListener(eventName, listener)`；将监听事件添加到最前面，但只监听一次。
`emitter.removeAllListeners(eventName, listener)`；移除所有的监听器。

```js
bus.removeAllListeners('click')
```

## 4. zlib 模块

`zlib` 模块提供了一系列的压缩和解压缩方法，用于处理 Node.js 中的 HTTP 请求与响应。

```js
const fs = require('node:fs')
const http = require('node:http')
const zlib = require('node:zlib')
const gzip = zlib.createGzip()
http.createServer((req, res) => {
  const rs = fs.createReadStream('./files/1.txt', 'utf-8')
  res.writeHead(200, {
    'Content-Type': 'text/html;charset=utf-8',
    'Content-Encoding': 'gzip'
  })
  rs.pipe(gzip).pipe(res)
}).listen(3000, () => {
  console.log('server is running')
})
```

## 5. crypto 加密

`crypto` 模块提供了一系列的加密方法，常用的有 MD5、SHA256 等哈希算法，以及 AES 等对称加密算法。

```js
const crypto = require('node:crypto')

// 哈希散列 —— createHash
const hash = crypto.createHash('sha256')
hash.update('hello world')
console.log(hash.digest('hex'))     // 按十六进制输出
console.log(hash.digest('base64'))  // 按 Base64 输出

// HMAC —— 带密钥的哈希
const hmac = crypto.createHmac('sha256', 'secret-key')
hmac.update('hello world')
console.log(hmac.digest('hex'))

// 对称加密（AES）
let key = '1234567890123456'  // 密钥，16 字节对应 aes-128
let iv = '1231231231231230'   // 初始化向量，16 字节
let data = 'coderyang111'     // 待加密数据

function encrypt(key, iv, data) {
  const cipher = crypto.createCipheriv('aes-128-cbc', key, iv)
  return cipher.update(data, 'utf8', 'hex') + cipher.final('hex')
}

function decrypt(key, iv, encrypted) {
  const decipher = crypto.createDecipheriv('aes-128-cbc', key, iv)
  return decipher.update(encrypted, 'hex', 'utf8') + decipher.final('utf8')
}

const encrypted = encrypt(key, iv, data)
const decrypted = decrypt(key, iv, encrypted)
console.log('加密:', encrypted)
console.log('解密:', decrypted)
```

## 7. 爬虫基础

1. 爬虫是使用 Node.js 爬取网页数据的一种方式，它通过发送 HTTP 请求获取网页数据，然后解析网页数据，获取需要的数据。
2. 有些请求返回的数据是 `JSON` 格式，可以直接解析，有些是 `HTML` 格式，需要使用 `cheerio` 库解析。

```js
const cheerio = require('cheerio')

function spider(data) {
  // data 为请求返回的数据，一般是html格式
  const $ = cheerio.load(data)
  // 例子 一些方法 获取网页中的数据数据
  let $moviesList = $(".column.content")
  let movies = []
  $moviesList.each((index, value) => {
    movies.push($(value).find(".title").text)
  })
  return JSON.stringify(movies)
}
