# Express 服务器

## 1. 了解 Express

1. 原生 `http` 模块构建 Web 服务器时，URL 判断、Method 判断、参数解析、逻辑处理都需要自己手动封装，较为繁琐。
2. Express 是目前 Node.js 最流行的 Web 框架之一，可以快速、方便地开发 Web 服务器，并通过中间件机制扩展功能。
3. 安装 Express：

```bash
npm init -y
npm i express
```

## 2. Express 基础使用

### 创建服务器

```js
const express = require('express')

const app = express()

// 监听 GET 请求 —— 路径 /
app.get('/', (req, res) => {
  res.send('Hello Express')
})

// 监听 POST 请求 —— 路径 /login
app.post('/login', (req, res) => {
  res.send('Welcome Back~')
})

// 开启端口监听
app.listen(8000, () => {
  console.log('Express 服务已启动 → http://localhost:8000')
})
```

### 中间件的概念

1. Express 本质是一个**中间件系统**。中间件就是一个函数，可以访问 `req`（请求对象）、`res`（响应对象）、`next`（下一个中间件函数）。
2. 当请求匹配到一个中间件后，**默认不会执行后续中间件**，除非调用了 `next()`。
3. Express 提供两种注册中间件的核心方式：

- `app.use([path], callback)`：匹配以 path 开头的所有请求方式和路径。
- `app.method(path, callback)`：如 `app.get`、`app.post`，匹配指定方法和路径。

### 普通中间件

```js
const express = require('express')
const app = express()

// 通过 next() 让请求流向下一个中间件
app.use((req, res, next) => {
  console.log('第 1 个普通中间件')
  next()
})

app.use((req, res, next) => {
  console.log('第 2 个普通中间件')
  next()
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

### 路径匹配中间件

```js
const express = require('express')
const app = express()

app.use((req, res, next) => {
  console.log('公共中间件——所有请求都会经过')
  next()
})

// 只有 /home 路径开头的请求才会匹配
app.use('/home', (req, res, next) => {
  console.log('home 路径中间件')
  next()
})

app.get('/home', (req, res) => {
  res.send('首页')
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

### 方法与路径同时匹配

```js
const express = require('express')
const app = express()

// 只有 GET /home 才会执行
app.get('/home', (req, res) => {
  res.send('首页')
})

// 只有 POST /login 才会执行
app.post('/login', (req, res) => {
  res.send('登录')
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

### 连续注册中间件

同一路径可以串联多个中间件，每个中间件都需调用 `next()` 才会继续：

```js
const express = require('express')
const app = express()

app.get('/home',
  (req, res, next) => {
    console.log('中间件 01：鉴权检查')
    next()
  },
  (req, res, next) => {
    console.log('中间件 02：日志记录')
    next()
  },
  (req, res) => {
    console.log('中间件 03：处理业务逻辑')
    res.send('首页内容')
  }
)

app.listen(8000, () => {
  console.log('服务已启动')
})
```

## 3. 请求参数解析

### URL 上的参数

```js
const express = require('express')
const app = express()

// 路径参数：/users/123
app.get('/users/:id', (req, res) => {
  console.log('路径参数:', req.params)  // { id: '123' }
  res.json(req.params)
})

// 查询参数：/search?keyword=node&page=1
app.get('/search', (req, res) => {
  console.log('查询参数:', req.query)   // { keyword: 'node', page: '1' }
  res.json(req.query)
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

### JSON 格式请求体

Express v4.16+ 内置了 JSON 解析中间件：

```js
const express = require('express')
const app = express()

// 解析 Content-Type: application/json 的请求体
app.use(express.json())

app.post('/login', (req, res) => {
  console.log(req.body)  // { username: 'admin', password: '123456' }
  res.json({ code: 0, message: '登录成功' })
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

### URL 编码格式请求体（x-www-form-urlencoded）

```js
const express = require('express')
const app = express()

// 解析 Content-Type: application/x-www-form-urlencoded
// extended: true   → 使用第三方库 qs 解析（支持嵌套对象）
// extended: false  → 使用 Node 内置 querystring 解析
app.use(express.urlencoded({ extended: true }))

app.post('/login', (req, res) => {
  console.log(req.body)
  res.json({ code: 0, message: '登录成功' })
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

### form-data 格式（文件上传）

**没有文件时**：

```js
const express = require('express')
const multer = require('multer')
const app = express()

// 不传配置，multer 只解析普通键值对，文件不处理
const upload = multer()

app.post('/login', upload.any(), (req, res) => {
  console.log(req.body)  // form-data 中的普通字段
  res.json({ code: 0, message: '登录成功' })
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

**有文件时**：

```js
const express = require('express')
const multer = require('multer')
const path = require('node:path')
const app = express()

// 配置文件存储
const storage = multer.diskStorage({
  destination: (req, file, cb) => {
    cb(null, './uploads/')
  },
  filename: (req, file, cb) => {
    // 用时间戳+原始扩展名防止重名
    cb(null, Date.now() + path.extname(file.originalname))
  }
})

const upload = multer({ storage })
// 注意：如果定义了 storage，就不要同时设置 dest，两者互斥

// 单文件上传 —— 使用 upload.single('字段名')
app.post('/avatar', upload.single('avatar'), (req, res) => {
  console.log('文件信息:', req.file)
  console.log('其他字段:', req.body)
  res.json({ code: 0, message: '上传成功' })
})

// 多文件上传 —— 使用 upload.array('字段名', 最大数量)
app.post('/photos', upload.array('photos', 10), (req, res) => {
  console.log('文件列表:', req.files)
  console.log('其他字段:', req.body)
  res.json({ code: 0, message: '上传成功' })
})

app.listen(8000, () => {
  console.log('文件上传服务已启动')
})
```

## 4. 响应结果设置

Express 对 `res` 进行了增强，提供了比原生 `http` 更丰富的响应方法：

```js
const express = require('express')
const app = express()

app.get('/user', (req, res) => {
  // 设置状态码
  res.status(200)

  // 设置响应头
  res.set('Content-Type', 'application/json')

  // send 可以自动设置 Content-Type（字符串 → text/html，对象/数组 → application/json）
  res.send({ name: 'yzb', age: 18 })

  // json 等同于 res.set('Content-Type', 'application/json') + res.send(data)
  res.json({ name: 'yzb', age: 18 })
  res.json(['a', 'b', 'c'])
})

app.get('/error', (req, res) => {
  // 链式写法
  res.status(404).json({ error: 'Not Found' })
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

| 方法 | 用途 |
|------|------|
| `res.send(data)` | 自动推断 Content-Type，发送字符串/Buffer/对象/数组 |
| `res.json(data)` | 发送 JSON 格式响应 |
| `res.status(code)` | 设置 HTTP 状态码 |
| `res.set(field, value)` | 设置响应头 |
| `res.type(type)` | 设置 Content-Type |
| `res.redirect(url)` | 重定向（默认 302） |
| `res.sendFile(path)` | 发送文件 |

> **注意**：Express 中应使用 `res.send()` 或 `res.json()` 来结束响应，不要使用原生 `res.end()`（`res.end()` 不会自动设置 Content-Type，且只能发送字符串/Buffer）。

## 5. Router 模块化路由

当项目变大时，不应把所有路由都写在 `app.js` 中。Express 提供了 `Router` 来实现模块化路由。

**主文件 `index.js`**：

```js
const express = require('express')
const usersRouter = require('./router/users')
const uploadRouter = require('./router/upload')

const app = express()

app.use(express.json())
app.use(express.urlencoded({ extended: true }))

// 路由前缀会拼接到子路由的路径前面
app.use('/users', usersRouter)
app.use('/upload', uploadRouter)

app.listen(8000, () => {
  console.log('服务已启动')
})
```

**路由文件 `router/users.js`**：

```js
const express = require('express')
const router = express.Router()

router.get('/', (req, res) => {
  res.json({ users: [] })
})

router.get('/:id', (req, res) => {
  res.json({ id: req.params.id })
})

router.post('/', (req, res) => {
  res.json({ code: 0, message: '创建成功' })
})

module.exports = router
```

**路由文件 `router/upload.js`**：

```js
const express = require('express')
const multer = require('multer')
const path = require('node:path')
const router = express.Router()

const storage = multer.diskStorage({
  destination: (req, file, cb) => cb(null, './images/'),
  filename: (req, file, cb) => cb(null, Date.now() + path.extname(file.originalname))
})

const upload = multer({ storage })

router.post('/', upload.array('file', 10), (req, res) => {
  console.log('上传的文件:', req.files)
  console.log('其他字段:', req.body)
  res.json({ code: 0, message: '上传成功' })
})

module.exports = router
```

## 6. 静态资源服务

```js
const express = require('express')
const app = express()

// 将 build 目录作为静态资源根目录
// 访问 http://localhost:8000/index.html 会映射到 ./build/index.html
app.use(express.static('./build'))

// 可以挂载多个静态目录
app.use('/public', express.static('./public'))

app.listen(8000, () => {
  console.log('静态资源服务已启动')
})
```

## 7. 错误处理中间件

Express 的错误处理中间件有**4 个参数** `(err, req, res, next)`，Express 通过参数数量来区分普通中间件和错误处理中间件：

```js
const express = require('express')
const app = express()

// 定义错误码常量
const ERR_USER_EXISTS = 'USER_ALREADY_EXISTS'
const ERR_USER_NOT_FOUND = 'USER_NOT_FOUND'

app.post('/register', (req, res, next) => {
  const isExists = true
  if (isExists) {
    // 在 next 中传入参数，Express 会自动跳过普通中间件，进入错误处理中间件
    return next(new Error(ERR_USER_EXISTS))
  }
  res.json({ code: 0, message: '注册成功' })
})

// 错误处理中间件 —— 必须有 4 个参数
app.use((err, req, res, next) => {
  let status = 400
  let message = ''

  switch (err.message) {
    case ERR_USER_EXISTS:
      status = 409
      message = '用户名已存在'
      break
    case ERR_USER_NOT_FOUND:
      status = 404
      message = '用户不存在'
      break
    default:
      status = 500
      message = '服务器内部错误'
  }

  res.status(status).json({
    errCode: status,
    errMessage: message
  })
})

app.listen(8000, () => {
  console.log('服务已启动')
})
```

## 8. Express 常用中间件速查

| 中间件 | 功能 | 安装 |
|--------|------|------|
| `express.json()` | 解析 JSON 请求体 | 内置 |
| `express.urlencoded()` | 解析 URL 编码请求体 | 内置 |
| `express.static()` | 托管静态资源 | 内置 |
| `multer` | 解析 multipart/form-data | `npm i multer` |
| `cors` | 处理跨域请求 | `npm i cors` |
| `compression` | Gzip 压缩响应 | `npm i compression` |
| `morgan` | HTTP 请求日志 | `npm i morgan` |
| `helmet` | 设置安全相关 HTTP 头 | `npm i helmet` |

使用示例：

```js
const express = require('express')
const cors = require('cors')
const compression = require('compression')
const morgan = require('morgan')
const helmet = require('helmet')

const app = express()

app.use(helmet())       // 安全头
app.use(cors())          // 跨域
app.use(compression())   // Gzip 压缩
app.use(morgan('dev'))   // 请求日志
app.use(express.json())

app.get('/api/data', (req, res) => {
  res.json({ hello: 'world' })
})

app.listen(8000)
```

## 9. Express 最佳实践

1. **使用 `res.json()` 而非 `res.end()`**：Express 的 `res.json()` 会自动设置 `Content-Type`。
2. **路由模块化**：用 `express.Router()` 拆分路由文件，保持代码结构清晰。
3. **统一错误处理**：用 4 参数的错误处理中间件集中管理错误响应。
4. **生产环境加安全中间件**：`helmet` 设置安全头，`cors` 控制跨域策略，`express-rate-limit` 限制请求频率。
5. **不要暴露 `X-Powered-By: Express`**：用 `app.disable('x-powered-by')` 关闭。
6. **始终处理异步错误**：Express 4.x 不会自动捕获 Promise 拒绝，需手动 `try/catch` 或使用 `express-async-errors` 包。
7. **使用 `compression` 中间件**：开启 gzip 压缩可大幅减小响应体积。
8. **环境区分**：用 `NODE_ENV` 环境变量控制不同环境的行为（日志级别、错误信息是否详细等）。
